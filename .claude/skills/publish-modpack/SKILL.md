# Publish Modpack

Release a new version of the Prism Launcher modpack to disqt.com/minecraft using the release CLI.

## Prerequisites

Warn the user: **Minecraft must be closed** before running. Locked files cause incomplete zips.

## Steps

1. **Ask the user for the version number** (e.g. "2.11")

2. **Ask for the Prism instance path** or default to the latest instance in `C:\Users\leole\AppData\Roaming\PrismLauncher\instances\`

3. **Check .packignore exists** in the instance root. If missing, warn the user -- without it, runtime dirs (saves, screenshots, DH data, logs) will bloat the zip.

4. **Run the release script**
   ```bash
   python C:\Users\leole\Documents\code\minecraft\prism-modpack-releaser\modpack_release.py "<instance-path>" --version "<version>"
   ```
   The script handles: zip, changelog (auto-generated from mod diffs), upload, manifest, symlink, prune, Discord notification.
   It will prompt for confirmation before publishing.

5. **Update CLAUDE.md** -- update the "Current modpack" line in `./CLAUDE.md` to reflect the new version.

6. **Report** the result (version published, changelog, Discord notification status).

## Options

- `--dry-run` -- preview without publishing
- `--no-notify` -- skip Discord notification
- `--keep N` -- override how many versions to retain (default: 3)

## Notes

- Changelog is auto-generated by diffing mod jars and configs against the previous VPS version. If Disquests was updated, it pulls release notes from GitHub.
- The DisqtVersion server plugin refreshes from the manifest URL every 30 minutes automatically -- no reload needed.
- Config file: `C:\Users\leole\Documents\code\minecraft\prism-modpack-releaser\modpack-release.json` (not committed, contains webhook secret).
